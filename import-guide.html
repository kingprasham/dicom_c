DICOM Recursive Import Guide

Learn how to import DICOM files from multiple subdirectories

Key Features
Recursive Scanning

Automatically scans all subdirectories and nested folders

Smart Detection

Identifies DICOM files by DICM marker or .dcm extension

Any Structure Supported

Works with any directory organization - flat or deeply nested

1 Understanding Directory Structures

The system supports ANY directory structure. Here are examples:

Single Series (Works)
C:\Users\prash\Downloads\dicom\series-000001\ ├── image-00001.dcm ├── image-00002.dcm └── image-00003.dcm
Multiple Series (Works)
C:\Users\prash\Downloads\dicom\ ├── series-000001\ │ ├── image-00001.dcm │ ├── image-00002.dcm │ └── image-00003.dcm ├── series-000002\ │ ├── image-00001.dcm │ └── image-00002.dcm ├── patient-john-doe\ │ ├── ct-chest\ │ │ └── scan.dcm │ └── mri-brain\ │ └── scan.dcm └── deeply\ └── nested\ └── folder\ └── study.dcm
2 Correct Path Format
Correct Paths (Use These)
C:\Users\prash\Downloads\dicom\ ✓ Parent folder with multiple series
C:/Users/prash/Downloads/dicom/ ✓ Forward slashes also work
C:\Users\prash\Downloads\dicom\series-000001\ ✓ Single series folder
Common Mistakes
\\server\share\dicom ✗ UNC paths may be slow - map network drive first
dicom\ ✗ Must be absolute path, not relative
3 Step-by-Step Import Process
  1. Login as Admin and navigate to Patients page
  2. Click "Hospital Config" button
  3. Scroll to "Import Existing Studies" section
  4. Enter the parent directory path:
    C:\Users\prash\Downloads\dicom\
  5. Click "Scan Directory" button
  6. System will display all DICOM files found in all subdirectories
  7. Review the list and click "Start Import"
  8. Monitor progress bar
  9. Go to Patients page to see imported studies
Troubleshooting
"No DICOM files found"
  • Verify directory path is correct
  • Check files have .dcm extension or DICM marker
  • Ensure PHP has read permissions
  • Run the test script to diagnose the issue
"Files imported but not showing"
  • Check Orthanc is running: http://localhost:8042
  • Click "Refresh" button on Patients page
  • Check database tables: cached_patients, cached_studies
